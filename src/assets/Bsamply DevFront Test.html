Movie Reviews

Description

The goal of this test is to use the Movie reciews API to implement the current html page with an Angular application (you will need to create an account to get an API key).

The user will be able to search via the input, filter by critic's pick and order either by publication date or by opening date.

By default 10 reviews will be loaded and the user will be able to load more by clicking the button bellow until there is no more reviews to load.

Each item will have the link to the full review (which will point to NYT itself).

Notes

your work will be evaluated depending to this cretaria:

  • if your app is functional in the right way without errors
  • how you structured your code and how you devided the project into components (think of it as scalable project, that will get bigger in the future)
  • the readability and the maintainability of your code (like if you are working in a team)
  • How much you care about details

Bonuses

the following notes are optional, but we recommend them:

  • This app is not responsive right now it will be great if you make it responive using tailwindcss.
  • We recommend you to keep the working history of you progress by keeping the commit history when you push the project
  • If you see that any of the asked features or components can be done in deffirent ways, you can justify your choosen solution by writing comments inside your code or creating a readme or pdf file.

Finally

When you finish working on the project, please push it to any git repository of your choice and give access to essam-ayari@dedagroup.it

If you have any notes or questions about the test, please don't hesitate to contact me via the same email.
